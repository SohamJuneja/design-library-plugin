# Colors

> Defines the palette for consistent use of color.

**Category:** Patterns

### Dos and Don'ts

| Do | Don't |
|---|---|
| Use color to draw attention to important parts of your interface. | Color shouldn't be the only way a piece of information is conveyed, use text or symbols along with it. |
| Use colors provided by Jenkins as they will automatically adapt to the user's theme. |  |

## Semantic

| Name | Description | CSS Class | CSS Variable |
|---|---|---|---|
| Accent | Use for primary actions. | `jenkins-!-accent-color` | `var(--accent-color)` |
| Text | Use for text. | `jenkins-!-text-color` | `var(--text-color)` |
| Secondary text | Use for secondary text. | `jenkins-!-text-color-secondary` | `var(--text-color-secondary)` |
| Success | Use for success states. | `jenkins-!-success-color` | `var(--success-color)` |
| Warning | Use for warning states. | `jenkins-!-warning-color` | `var(--warning-color)` |
| Error | Use for error states. | `jenkins-!-error-color` | `var(--error-color)` |
| Build | Use for build kickoff. | `jenkins-!-build-color` | `var(--build-color)` |
| Destructive | Use for destructive actions. | `jenkins-!-destructive-color` | `var(--destructive-color)` |
